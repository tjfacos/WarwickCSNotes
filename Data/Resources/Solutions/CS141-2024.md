```hs

```
Your implementation should have the same semantics as the definition in the Haskell standard library. (This function is **not** included in the Appendix.)

> [!hint]- Hint
> For each element `x` in a list, if `x` is `Nothing`, `catMaybes` discards it. If `x` is `Just y`, then `catMaybes` replaces it with `y`

> [!check]- Solution
> ```hs
> -- Version using explicit recursion
> catMaybes :: [Maybe a] -> [a]
> catMaybes [] = []
> catMaybes (Nothing  : xs) =     catMaybes xs
> catMaybes ((Just y) : xs) = y : catMaybes xs
> 
> -- Version using implicit recursion (map & filter)
> catMaybes :: [Maybe a] -> [a]
> catMaybes = (map $ \(Just x) -> x) 
>     . (filter $ \x -> case x of Just y -> True; Nothing -> False) 
> 
> -- An example
> >>> catMaybes [Just 5, Nothing, Just 6, Just 7, Nothing, Nothing]
> >>> [5, 6, 7]
> ```

> [!check]- Solution
> ```hs
> filter :: (a -> Bool) -> [a] -> [a]
> filter f = catMaybes . (map $ \x -> if f x then Just x else Nothing)
> ```

# Question 3 - Polymorphism & Type Classes
**(a)** Briefly describe what is meant by the _type class hierarchy_, with reference to a suitable example from the Haskell standard library.

> [!check]- Solution
> Some type classes, for practical or semantic reasons, are subclasses of another class. This means that every member of that class must also be a member of the other class. An example is `Functor`, `Applicative`, and `Monad`: All `Monad`s are `Applicative`s, and all `Applicative`s or `Functor`s.

**(b)**  Very briefly outline two reasons why one might make a type class B a subclass of type class A.

> [!check]- Solution
> 1. To allow values of type **B** to be used with functions and operators defined in type **A**
> 2. For semantics - it sometimes does not make sense for an instance of **B** to exist without an instance of **A** (for instance, having the notion of an Integral without the notion of a Number).

In mathematics, a group is a set S equipped with:
* An associative binary operation $\circ$;
* An identity value I such that $(x \circ I) = (I \circ x) = I$ for all $x \in S$;
* For each value $x \in S$, an inverse $x^{−1}$ such that $(x^{−1} \circ x) = (x \circ x^{−1}) = I$.

**(c)** Define a type class, `Group`, that is a subclass of `Monoid`, and provides the function inverse.

> [!check]- Solution
> ```hs
> class Monoid g => Group g where
>     inverse :: g -> g
> ```

The wrapper type `Sum` is defined in the Haskell standard library as follows:

```hs
newtype Sum a = Sum { getSum :: a }
```

**(d)** Give a polymorphic instance of the Group type class for `Sum a`, where `a` is known to be a member of `Num`. You may assume that a suitable `Monoid` instance is already defined.

> [!check]- Solution
> ```hs
> instance Num a => Group (Sum a) where
>     inverse (Sum x) = Sum (negate x)
> ```

**(e)** Prove that your definition given in part (d) fulfils the given mathematical definition of inverse for the type Sum Integer. You may make use of any other facts that you know about the Sum type.

> [!check]- Solution
> 1. `Semigroup` (superclass of `Monoid`) provides us the binary operation `(<>)`. For `Sum Integer`, `(<>)` adds together the wrapped values
> 2. `Monoid` provides an identity value `mempty` (`= Sum { getSum = 0 } :: Sum Integer`)
> 3. For any integer `x`, `x + (-x) = 0`, so `(Sum x) <> inverse (Sum x) = mempty`

A _matching_ is a set of relationships between pairs of values that “map” to each other,
such that no element appears in more than one matching.

**(a)** Define a `Matching` data type which will be used to represent a matching. Define the above matching (see paper) as a value of type `m :: Matching Char Int`

> [!check]- Solution
> ```hs
> newtype Matching a b = M { getMatchings :: [(a, b)] }
> 
> m :: Matching Char Int
> m = M [('a', 1), ('b', 4), ('c', 2)]
> ```

**(b)** Define a pair of functions
```hs
lookupR :: a -> Matching a b -> Maybe b
lookupL :: b -> Matching a b -> Maybe a
```
which return a value and return the value it is paired with, if such a pair exists. Add any needed type class constraints to these functions.

> [!check]- Solution
> ```hs
> lookupR :: Eq a => a -> Matching a b -> Maybe b
> lookupR x (M l) = lookup x l
> 
> lookupL :: Eq b => b -> Matching a b -> Maybe a
> lookupL x (M l) = lookup x $ map (\(x,y) -> (y,x)) l
> ```

**(c)** Define a function
```hs
insert :: a -> b -> Matching a b -> Matching a b
```
which adds a new mapping to the given `Matching`. If either value is in the
`Matching` already, the new mapping should replace the old ones.

> [!check]- Solution
> ```hs
> insert :: (Eq a, Eq b) => a -> b -> Matching a b -> Matching a b
> insert x y (M l) = let
>     l'  = filter (\(a,b) -> a == x) l
>     l'' = filter (\(a,b) -> b == y) l'
>     in (x,y) : l''
> ```

**(d)** Under what condition(s) can a function of type `a -> b` be represented as a value in your `Matching` type?

> [!check]- Solution
> The function must be **injective**, because a values on the right cannot be matched to two values on the left.

**(e)** Write a function
```hs
fromFunction :: Enum a, Bounded a
    => (a -> b)
    -> Maybe (Matching a b)
```
that takes a function `f` and returns a Matching that maps each value of type `a` to
`f a`. If the function cannot be represented as a `Matching`, then `fromFunction`
should return `Nothing`. [5]
_Hint_: The `Enum` and `Bounded` type classes will help you to get all the values of
type `a`.

> [!check]- Solution
> ```hs
> fromFunction :: Enum a, Bounded a => (a -> b) -> Maybe (Matching a b)
> fromFunction f = let
>         pairs = [(x, f x) | x <- [minBound .. maxBound]]
>         -- For reference 
>         -- foldr :: ((a, b) -> Matching a b -> Matching a b) 
>         --     -> Matching a b -> [(a,b)] -> Matching a b
>         matching = foldr (\(x, y) -> \m -> insert x y m) (M []) pairs
>     in if length $ getMatchings matching == length pairs then Just matching else Nothing
> ```

# Question 5 - Functors, Applicatives, and Monads

_Note_: An instance is "suitable" if it satisfies all relevant type-class laws, and includes all required functions. It may not be automatically derived. You do not need to prove that your instances are law-abiding unless asked to do so.

Consider the so-called "identity functor", defined as follows, that can wrap any value of type `a`.

```hs
newtype Identity a = Identity a
```

**(a)** Define a suitable `Functor` instance for `Identity`. Show that your instance satisfies the functor fusion law (given in the Appendix).

> [!hint]- Hint
> The functor fusion law states: $\text{fmap } (f \circ g) = \text{fmap } f \circ \text{fmap } g$

> [!check]- Solution
> ```hs
> instance Functor Identity where
>     -- fmap :: (a -> b) -> Identity a -> Identity b
>     fmap f (Identity x) = Identity (f x)
> 
> -- Proof
> fmap (f . g) $ Identity x = Identity ( (f.g) x ) 
>     = Identity ( \y -> f(g y) $ x ) = Identity ( f(g x) )
> 
> (fmap f) . (fmap g) $ Identity x = \y -> fmap f (fmap g y) $ Identity x
>     = fmap f (fmap g Identity x) = fmap f $ Identity (g x) = Identity ( f(g x) )
> ```

**(b)** Define a suitable `Applicative` instance for `Identity`.

> [!check]- Solution
> ```hs
> instance Applicative Identity where
>     -- pure :: a -> Identity a
>     pure x = Identity x
>     -- (<*>) :: Identity (a -> b) -> Identity a -> Identity b
>     (Identity f) <*> (Identity x) = Identity (f x)
> ```

**(c)** Define a suitable `Monad` instance for `Identity`.

```hs
instance Monad Identity where
    -- (>>=) :: Identity a -> (a -> Identity b) -> Identity b
    (Identity x) >>= f = f x
```

**(d)** Define a suitable `Functor` instance for `Compose`

> [!hint]- Hint
> I suggest breaking down the types, and seeing how they'll have to fit together over `fmap`.

> [!check]- Solution
> ```hs
> instance (Functor f, Functor g) => Functor (Compose f g) where
>     -- For g, fmap :: (a -> b) -> g a -> g b

>     -- Breaking down the above
>         -- fmap h :: g a -> g b
>         -- fmap (fmap h) :: f (g a) -> f (g b)
> ```

    -- We need to lift the given value x :: a into g to get x' :: g a, then lift x' into f
    pure x = Compose $ pure $ pure x

    -- We have p :: f (g (a -> b)) and x :: f(g a)

        -- fmap (<*>) :: f (g (a -> b)) -> f (g a -> g b)
        -- So fmap (<*>) p :: f (g a -> g b)
        -- And finally (fmap (<*>) p) <*> x :: f (g b)
